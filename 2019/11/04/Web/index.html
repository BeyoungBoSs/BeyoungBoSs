浏览器内核、渲染引擎、JS引擎简介

定义

  • 浏览器内核分成两部分：渲染引擎和 jS引擎

  • 由于JS引擎越来越独立，浏览器内核 就倾向于 单指 渲染引擎

    常见的一些引擎

    渲染引擎

  • firefox使用gecko引擎

    gecko引擎
  1. Mozilla 现已面向 Android 的全新移动浏览器 FirefoxPreview。与大多数浏览器所采用的 Bink 渲染引擎不同，Firefox Preview 的渲染引擎 GeckoView 由 Mozilla 自己开发，这种独立性一定程度上避免了互联网被单一的科技巨头控制
    Bink 渲染引擎

Bink 是目前 Windows 游戏中应用最广泛的动画格式. 它具有压缩比高, 画质好, 对回放系统要求低等优点.许多游戏开场都采用该格式进行播放，此格式还具有不能轻易被他人转码的优点，非常棒的画面以及可以集成内置播放器受到了众多视频制作者的青睐。Bink Video是一种具有高压缩比、高画面质量、高音频质量，更优于MPEG2编码格式的优秀视频文件格式。 Bink Video 文件以 bik 为后缀名，被广泛的运用于计算机电子游戏 (Computer Electronic Game) 上，具体表现为开场电影(动画)、任务剧情电影(动画)，以及版权声明动画等等。很少有媒体播放工具支持 Bink Video 格式的文件，因为缺乏解码系统。 游戏中播放 Bink Video 通常会使用游戏自带的解码系统，或是自行研发，或是采用官方发布版本。BaNaNa Binkplay 采用官方发布的解码系统，可以完美的播放出 Bink Video 文件。使用 BaNaNa Binkplay ，您可以轻松的在非游戏状况下观看这些激动人心的电影(动画)。BaNaNa Binkplay 构建于 32 位 Windows 操作平台，能够在 Windows 95+SP3/95 OSR2/98/98 SE/ME/2000/XP以及以后的 Windows 版本上运行。

我们都知道 Android 系统提供了内置的 WebView，应用程序可通过 hook 的方式以在 app 的上下文中显示网页。不过 Android 的 WebView 并非真正用于构建浏览器，所以它禁用了许多高级 Web API。另外，它的目标平台也不是统一的，不同的可能有不同版本的 WebView

Hook

微软的MSDN中，对Hook的解释为：
A hook is a point in the system message-handling mechanism where an application can install a subroutine to monitor the message traffic in the system and process certain types of messages before they reach the target window procedure.

微软只是简单的将Hook解释为一种过滤（或叫挂钩）消息的技术。
我们这里讲解的Hook，简单解释为：挂钩，挂钩一切事物。包含微软的解释。
挂钩的事物通常指的是函数。
Hook 目的:过滤一些关键函数调用，在函数执行前，先执行自己的挂钩函数。达到监控函数调用，改变函数功能的目的。

  • opera最早使用Presto引擎，后来弃用

  • chrome\safari\opera使用webkit引擎

  • 13年chrome和opera开始使用Blink引擎
